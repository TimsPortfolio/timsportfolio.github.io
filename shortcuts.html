Shortcuts

Welcome to my shortcuts page. This is where I will document all of the weird, fun, and sometimes useful shortcuts.

What are shorcuts?

Shorcuts are automations that will preform a series of tasks once you trigger it. This is useful for automating repetative tasks, but can also be used to create fun programs. Most of the shorcuts that are mentioned here are not used to automate a task but to do a completely new job that probably wasn't needed in the first place.

My shortcuts

Verification - 7/28/20

This applet uses both Apple Shortcuts and IFTTT. It is just a simple identity verification app that uses your username and password, then it has an email and phone call verification. It starts with Apple Shortcuts by taking your entered email and password, hashing them and checking them against a stored hash. It then generates a random 6 digit number and emails it. This number has to be typed in as well. Then a random 4 digit number is generated, and sends an email to IFTTT to call you with the 4 digit number. You have to type this number in as well. It then checks if you failed any of the verification steps. If you did, then it will say that you failed verification, but it won't tell you which steps you failed. You just have to start over. Otherwise it will tell you that you passed. Right now the verification is protecting anything, and the email and phone call is being sent to the same phone that you are doing the verification on, but it is still fun to see it's possible.

App Logger

This applet uses Apple Shortcuts. It is actually created with many different shortcuts working together. It logs what apps are opened, then if the app cateogry is blacklisted it can leave the app and either show an alert, lock the phone, or play an alarm. The log can log different things, including location, a photo, and when the phone is unlocked. A detailed explenation of how it works is shown below.
Whenever an app is opened, it triggers an automation. There are many different automations for different app cateogries, including games and education. These automations don't really log or block anything, they just relay information to the shortcuts with what cateogry was opened. The shortcut will first figure out if the selected category is blocked. If it isn't, then it will just log the app that is opened, then stop. If it is blocked, it will see what settings are selected. It can just block the app, lock the phone when the app is opened, and play an alarm. The log will by default just log the time and app opened in a text file. It can also log the location and send emails.
As mentioned before, there are many different settings. This includes what app cateogries are blocked, email notifications, location information, picture from the front camera, locking the phone, sounding the alarm, and the volume of the alarm. When the settings are opened, it will first ask for a pin. If the pin is correct, it will show a menu with the options, and what the current state of that option is. If you select an option, it will run a shortcut which is responsable for changing the text file from 0 to 1, and vise versa. The settings can also be changed on a mac because all the text files are synced with the cloud.
